# fix-korean-prose

이미 존재하는 한국어 텍스트의 **문체만** 교정한다. 리서치·사실검증·문서 생성은 하지 않는다 —
입력의 사실·수치·고유명사·코드·인용 `[n]`을 보존하면서 표현만 자연스럽게 다듬는다.
새 문서 생성이 필요하면 `write-korean-docs`(무거운 파이프라인)로 가라.

## 1. 입력 감지

- 인자가 **존재하는 `.md` 파일 경로**면 → 파일 모드. 그 파일을 Read한다.
- 그 외(붙여넣기 텍스트·문장)면 → 텍스트 모드. 받은 텍스트를 교정 대상으로 삼는다.
- 빈 입력·존재하지 않는 경로면 → 무엇을 교정할지 되묻고 중단한다(조용한 실패 금지).

## 2. 교정 (PROSE_RULES)

아래 기준으로 교정본을 만든다:

- S1(무조건 제거): em dash(—), 이모지, 산문 속 세미콜론, 이중 피동(되어진다·보여지다·잊혀지다 등 -여/혀/려/겨지다 계열).
- 번역투: "~에 의해 / ~를 통해 / ~에 대해", 과한 명사화("~에 대한 처리를 수행")를 동사형으로 푼다. 일본어식 쉼표 과다(주제격 "는/은/도" 뒤 쉼표)와 관형격 "-의" 연쇄 남용을 줄인다.
- 만연체 회피: 한 문장에 절을 겹겹이 내포하지 말고 끊는다. 핵어-의존어 거리와 내포 깊이를 줄여 독자의 처리 부하를 낮춘다.
- 존댓말 어미는 한 문서에서 한 등급으로 일관되게 유지한다.
- 가능성 표현("~할 수 있다") 남발을 단정형으로 바꾼다.
- 영어 관습 전이 회피: 불필요한 주어(우리는/이것은), rule of three 강박, "단순히 X가 아니라 Y" 대구.
- 사실·수치·고유명사·코드 블록·인용 표기 `[n]`은 100% 보존한다. 날조 금지 — 근거 없으면 채우지 말고 삭제한다.

## 3. S1 결정론 검증 (Bash 강제)

교정본을 임시 파일에 쓴 뒤 CLI로 S1을 검사한다. node 실행 전 nvm을 로드한다:

```bash
export NVM_DIR="$HOME/.nvm"; . "$NVM_DIR/nvm.sh"; nvm use
node "$CLAUDE_PLUGIN_ROOT/lib/prose-cli.js" <교정본_파일.md>
```

- exit 0 = S1 clean. exit 1 = 위반 있음(stdout JSON의 `s1[]`에 위반 종류·위치).
- **위반이 있으면 그 종류를 제거하도록 재교정**하고 다시 검사한다(폐루프).

## 4. 자연스러움 리뷰 + redo 폐루프 (상한 2회)

S1이 clean이 된 뒤, "이 글이 왜 아직도 AI처럼 읽히나"를 스스로 리뷰한다 — 균일한 리듬,
의견 부재, 기계적 전환어, 남은 번역투. 지적이 있으면 재교정하고 §3의 S1 검사를 다시 통과시킨다.

- 재교정은 **최대 2회**까지. 2회 후에도 S1 잔여가 있으면 남은 위반을 "남은 S1 N건: <목록>"으로
  보고하고 사용자에게 판단을 위임한다(무한 재시도 금지).
- 재교정이 사실·인용 `[n]`을 훼손하면 그 재교정을 버리고 직전본을 유지한다.

## 5. 출력

- **텍스트 모드** → 교정본을 응답에 표시한다. 파일을 쓰지 않는다.
- **파일 모드** → 원본 대비 diff를 보여준다. **원본 덮어쓰기는 사용자가 명시로 확인한 뒤에만** 한다
  (비가역 변경 보호). 확인 전에는 원본을 건드리지 않는다.
