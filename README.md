# korean-docs

한글 기술 레퍼런스 문서(`.md`)를 만들고 고치는 Claude Code 워크플로우 플러그인. [deep-research](https://docs.claude.com/en/docs/claude-code)를 오마주해, 주제 하나를 받아 리서치부터 한국어 문체 교정까지 한 번에 처리한다. 기존 문서를 받아 개정하는 편집 모드도 있다.

핵심은 두 가지다. 사실은 **적대적으로 검증**하고(출처가 주장을 실제로 뒷받침하는지 3표로 가린다), 문체는 **한국어답게** 만든다(번역투와 AI슬롭, em dash, 이중 피동을 결정론적으로 잡는다).

## 무엇이 다른가

- **deep-research와 달리** 출력이 영어 리서치 리포트가 아니라 한국어 기술 레퍼런스다. 5·6단계에 한국어 문체 품질 게이트(번역투·AI슬롭 제거, 존댓말 일관성)를 둔다.
- **기계적 humanizer(im-not-ai 등)와 달리** 다듬기만 하지 않고 리서치·생성까지 한다. 문체 규칙은 그 계보(한국 번역학 + KatFishNet)에서 가져왔다.
- **외부 에이전트·스킬 의존이 없다.** 워크플로우는 단일 자기완결 파일이라 설치가 파일 하나 복사로 끝난다.

## 동작 방식

워크플로우는 7단계 파이프라인이다.

```
스코프 → 리서치 → 사실 검증 → 초안 → 문체 교정 → 자연스러움 검증 → 조립
```

1. **스코프**: 주제에서 독자·톤·섹션 구조와 섹션별 리서치 질문을 설계한다.
2. **리서치**: 섹션마다 웹·공식 문서를 조사해 출처가 달린 사실을 모은다.
3. **사실 검증**: 사실 하나하나를 검증자 3명이 적대적으로 가린다. 출처가 주장을 뒷받침하지 못하면 기각, 불확실하면 기각이 기본값이다. 2표 이상 통과해야 확정.
4. **초안**: 확정된 사실만으로 섹션을 쓴다. 사실에 없는 내용은 추가하지 않고, 부족하면 '출처 필요'로 표시한다(날조 금지).
5. **문체 교정**: 번역투·AI슬롭을 한국어답게 고친다.
6. **자연스러움 검증**: "아직도 AI처럼 읽히는가"를 적대적으로 되묻고, 걸리면 다시 쓴다.
7. **조립**: 섹션을 합치고 인용 번호와 출처 목록을 붙인다.

### 품질 게이트

생성물이 그럴듯한 거짓이나 AI 티로 새지 않게 세 곳을 닫는다.

- **닫힌 루프**: 자연스러움 검증은 검사 → 수정 → **재검사**다. 재작성본을 검증 없이 내보내지 않고, 상한(`MAX_REDO`)까지 다시 판정한다.
- **결정론 게이트와 LLM 판정의 AND**: em dash·이모지·산문 속 세미콜론·이중 피동(`되어진다`·`보여지다` 등)은 정규식 0/1 하드 게이트로 잡고, LLM의 자연스러움 판정과 함께 통과해야 섹션이 확정된다.
- **빈 껍데기 드롭**: 확정된 사실이 0건인 섹션은 초안 전에 버린다. 작가에게 빈 섹션을 넘기면 '출처 필요'만 가득한 산문이 나오기 때문이다.

## 설치

마켓플레이스 설치를 권장한다.

```
/plugin marketplace add HarryJhin/korean-docs
/plugin install korean-docs@korean-docs-marketplace
```

설치하면 `write-korean-docs` pre-flight 스킬이 등록되고, SessionStart 훅이 워크플로우를 `~/.claude/workflows/`로 복사한다. **다음 세션부터** `/korean-docs` 슬래시 커맨드가 생긴다.

> 워크플로우는 아직 1급 플러그인 컴포넌트가 아니라(plugin.json에 `workflows` 필드 없음, [anthropics/claude-code#66032](https://github.com/anthropics/claude-code/issues/66032)) 훅으로 설치한다. 수동 복사 방법은 [INSTALL.md](./INSTALL.md)를 참조한다.

## 사용

### 생성: 주제에서 새 문서

`/korean-docs`에 주제를 자연어로 넘긴다(`/deep-research`와 같은 방식). 옵션 폼(JSON)을 채울 필요가 없다.

```
/korean-docs JavaScript Array.prototype.flat() 메서드 레퍼런스
```

독자·톤을 정하고 싶으면 별도 필드로 빼지 말고 주제 문장에 그대로 녹인다. 아웃라인 단계가 알아서 추출한다.

```
/korean-docs JavaScript Array.prototype.flat() 레퍼런스. 독자는 flat을 처음 쓰는 프론트 개발자, 톤은 간결한 레퍼런스체.
```

pre-flight 스킬을 깔았다면 "한글 레퍼런스 문서 만들어줘" 같은 자연어로도 진입한다. 주제가 모호하면 한두 가지만 되묻고, 실행 전에 비용을 고지한다.

### 편집: 기존 문서 개정 (v0.2.0+)

이미 있는 기술 레퍼런스 `.md`를 받아 **구조를 다시 짜고 내용을 갱신한다**. 다듬기만 하는 게 아니라 전면 개정이다.

pre-flight 스킬에 파일 경로와 함께 편집 의도를 주면 된다.

```
이 문서 개정해줘: docs/array-flat.md
```

스킬이 그 파일을 읽어 본문을 `existingDoc` 인자로 워크플로우에 넘긴다(파일 본문은 길어서 슬래시 커맨드 문자열에 실리지 않으므로, 읽어서 인자로 싣는 건 스킬의 몫이다). 주제를 따로 주지 않으면 문서의 H1에서 가져온다.

편집 모드의 동작 원칙은 **재검증된 보존**이다.

- **구조는 자유롭게 재설계한다.** 이상적 레퍼런스 구조를 새로 설계하고, 기존 섹션은 매핑해 잇거나 합치거나 버린다.
- **검증을 통과한 원본 내용만 보존한다.** 원본의 사실 주장도 신규 리서치 사실과 똑같이 3표 검증을 거친다. 통과하면 살리고, 출처가 없거나 검증에 떨어지면(stale·오류) 버린다. "맞는 내용은 지키고, 틀린 내용은 도려낸다."
- **문체·인용을 정리한다.** em dash 같은 S1 위반을 고치고, 인용 번호는 새 출처 목록에 맞춰 다시 매긴다.

출력은 **원본을 덮어쓰지 않는다.** 개정본을 `{원본}.revised.md`로 따로 저장하거나 diff로 보여주고, 덮어쓰기는 사용자가 확인한 뒤에만 한다.

## 출력 검수

워크플로우가 끝나면 사람이 확인할 것:

1. 반환된 `markdown`을 파일로 저장한다.
2. 워크플로우 로그에서 드롭된 섹션 경고를 확인한다(누락 섹션 여부). 편집 모드면 "재사용/신규 카운트"와 "원본 주장 N/M 검증 통과" 로그로 무엇이 보존·탈락했는지 본다.
3. **사실 정확성과 출처 유효성은 사람이 검수한다.** 날조된 API·수치가 없는지, `[n]` 인용이 실제 출처와 맞는지 본다. 적대 검증을 거쳐도 최종 판정은 사람이다.

문체 점검은 라이브러리로 자동화할 수 있다.

```bash
node -e "import('./lib/prose-checks.js').then(m => \
  console.log(m.proseScore(require('fs').readFileSync('OUT.md','utf8'))))"
```

## 개발

```bash
nvm use            # Node 24 (.nvmrc)
node --test        # 라이브러리·구조 테스트
```

테스트는 두 종류다. `lib/`의 결정론 함수(문체 검사, 문서 인제스트)는 단위 테스트로 검증하고, 워크플로우는 소스를 정규식·`vm` 컴파일로 스캔하는 구조 테스트로 막는다. e2e는 실제 에이전트를 spawn해 유료이므로 자동 테스트 밖이고, 수동 스모크로 확인한다.

> 워크플로우 파일(`workflows/korean-docs.js`)은 단일 파일 자기완결이라 `import`를 쓸 수 없다. `lib/`의 공유 로직(S1 패턴, 인제스트)은 워크플로우에 인라인으로 복제하며, 한쪽을 고치면 양쪽을 맞춘다.

## 범위

현재는 Diátaxis '레퍼런스' 유형, 단일 `.md`만 지원한다(생성·편집 공통). 하우투·튜토리얼·설명 유형은 후속이다.

## 버전

권위 버전은 `.claude-plugin/plugin.json`의 `version`이다. `package.json`이 이를 미러링하고, 범프할 때마다 [CHANGELOG.md](./CHANGELOG.md)에 항목을 남기며 `v{version}` 태그를 단다. 변경 이력은 CHANGELOG를 참조한다.
